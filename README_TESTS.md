# Инструкция по запуску тестов

## Установка зависимостей

Перед запуском тестов необходимо установить все зависимости:

```bash
npm install
```

## Запуск тестов

### Базовый запуск
```bash
npm test
```

### Запуск в watch режиме (автоматический перезапуск при изменениях)
```bash
npm test -- --watch
```

### Запуск с UI интерфейсом
```bash
npm run test:ui
```

### Запуск с покрытием кода
```bash
npm run test:coverage
```

## Структура тестов

Тесты расположены рядом с тестируемыми файлами в папках `__tests__`:

- `src/hooks/__tests__/` - тесты для хуков
- `src/api/__tests__/` - тесты для API клиента
- `src/components/__tests__/` - тесты для компонентов

## Написанные тесты

### 1. useHotkeys.test.ts
Тестирует функциональность обработки горячих клавиш:
- Вызов обработчика при нажатии клавиши
- Игнорирование при вводе в input
- Разрешение клавиши "/" даже в input
- Отключение хука
- Обработка нескольких горячих клавиш
- Очистка при размонтировании

### 2. useTheme.test.ts
Тестирует управление темой:
- Инициализация со светлой темой
- Загрузка из localStorage
- Сохранение в localStorage
- Отправка события themechange
- Переключение темы

### 3. client.test.ts
Тестирует API клиент:
- Получение списка объявлений
- Преобразование категории в categoryId
- Получение объявления по ID
- Одобрение объявления
- Отклонение объявления
- Отправка на доработку
- Отмена запросов

### 4. AppHeader.test.tsx
Тестирует компонент шапки:
- Отображение заголовка
- Наличие ссылок навигации
- Отображение переключателя темы

## Добавление новых тестов

При добавлении новых компонентов или функций создавайте тесты в соответствующих папках `__tests__`.

Пример структуры:
```
src/
  components/
    MyComponent.tsx
    __tests__/
      MyComponent.test.tsx
```

## Моки и тестовые данные

Моковые данные для тестов находятся в `src/test/mockData.ts`.
